# Project Structure

Understanding the layout helps you navigate and customize the configuration.

## Directory Overview

```
shuredotnix/
├── flake.nix           # Main entry point
├── identity.nix        # User identity (username, locale, theme)
├── shells.nix          # Development shells
│
├── hosts/              # Machine-specific configs
│   └── desktop/        # Desktop workstation
│       ├── default.nix
│       └── hardware-configuration.nix
│
├── library/            # Reusable modules
│   ├── core/           # NixOS system modules
│   ├── display/        # Hyprland, Waybar, Stylix
│   ├── home/           # Home Manager modules
│   └── profiles/       # Host profiles (desktop, server)
│
├── users/              # User-specific configs
│   └── shure/
│       ├── home.nix    # Home Manager entry
│       └── nixos.nix   # NixOS user settings
│
├── pkgs/               # Custom packages
├── overlays/           # Package modifications
└── secrets/            # Encrypted secrets (SOPS)
```

## Key Directories

### `library/core/`

System-level NixOS modules:

| File | Purpose |
|------|---------|
| `audio.nix` | PipeWire audio configuration |
| `boot.nix` | Bootloader and kernel settings |
| `gpu/` | GPU drivers (NVIDIA, AMD) |
| `network.nix` | NetworkManager, firewall |
| `security.nix` | Security hardening |

### `library/display/`

Desktop environment configuration:

| File | Purpose |
|------|---------|
| `hyprland.nix` | Hyprland compositor |
| `waybar/` | Status bar |
| `stylix.nix` | System-wide theming |
| `sddm.nix` | Login manager |

### `library/home/bundlers/`

User applications grouped by type:

| Directory | Contains |
|-----------|----------|
| `cli/` | Shell, Git, CLI tools |
| `desktop/` | Browser, Discord, Editor |

## Adding a New Host

1. Create `hosts/myhost/default.nix`
2. Copy/generate `hardware-configuration.nix`
3. Build with `nixos-rebuild switch --flake .#myhost`

The flake auto-discovers hosts from the `hosts/` directory.
