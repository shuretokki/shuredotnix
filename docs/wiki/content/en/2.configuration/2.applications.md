# Applications

dotnix uses a **Bundler** system to manage applications. Instead of a giant list of packages, apps are grouped into logical "bundles" that include their configuration.

## concept

Bundles are located in `library/home/bundlers/`. Each bundle is a self-contained module that configures an application (or set of tools).

- **CLI Bundles**: `library/home/bundlers/cli/` (e.g., git, starship, bat)
- **Desktop Bundles**: `library/home/bundlers/desktop/` (e.g., browser, discord, vicinae)

## Adding Applications

To enable a bundle, import it in your user configuration:

```nix [users/shure/home.nix]
imports = [
  # Desktop Apps
  ../../library/home/bundlers/desktop/browser.nix
  ../../library/home/bundlers/desktop/discord.nix

  # CLI Tools
  ../../library/home/bundlers/cli/git.nix
  ../../library/home/bundlers/cli/starship.nix
];
```

## Creating a New Bundle

1. Create a new file in the appropriate bundler directory (e.g., `library/home/bundlers/cli/my-tool.nix`).
2. Write a standard Home Manager module:

```nix
{ pkgs, ... }:
{
  home.packages = with pkgs; [
    my-tool
  ];

  # Optional configuration
  programs.my-tool.enable = true;
}
```

3. Import your new bundle in `home.nix`.

## Available Bundles

### Desktop
- **Browser**: Zen Browser (customized)
- **Discord**: With Vencord extensions
- **Editor**: VS Code / Neovim
- **Terminal**: Alacritty / Kitty
- **Music**: Spotify / YouTube Music

### CLI
- **Shell**: Fish + Starship
- **Git**: Git with Delta diffs
- **Tools**: Bat, Eza, Fzf, Zoxide, Btop
